omicverse.fm.router¶

source module omicverse.fm.router

FM Router — LLM-based routing for single-cell foundation model tasks.¶

Takes a natural language query and returns:

  • Inferred task (embed / integrate / annotate / spatial / perturb / drug_response)
  • Selected best-fit model from :mod:omicverse.fm.registry
  • Resolved parameters (or clarifying questions)
  • Executable tool-call plan

Classes

  • RouterIntent — Inferred task intent from user query.

  • RouterInputs — Input information from user query.

  • ModelSelection — Model selection with rationale.

  • RouterSelection — Model selection output.

  • ResolvedParams — Resolved parameters for execution.

  • ToolCall — A single tool call in the execution plan.

  • Question — A clarifying question for the user.

  • RouterOutput — Complete router output.

Functions

  • validate_router_output — Validate router output against schema and registry.

  • build_model_cards — Build formatted model cards for LLM prompt.

  • build_router_prompt — Build the user prompt for the router LLM.

  • call_router_llm — Call the router LLM via context["_call_agent"].

  • route_query — Main router function — orchestrates profiling, LLM call, validation, rerouting.
