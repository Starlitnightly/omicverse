Cell-cell communication with CellPhoneDB#

Overview#

Cell-cell communication (CCC) analysis connects ligand-receptor activity with cell-state changes across complex tissues. This tutorial walks through CellPhoneDB-style communication analysis in OmicVerse, starting from the communication AnnData and then moving through heatmap, network, summary, and advanced visualization layers.

Citation#

Paper: Single-cell reconstruction of the early maternal-fetal interface in humans

Code: ventolab/CellphoneDB

CellChat Reference: Inference and analysis of cell-cell communication using CellChat

Run CellPhoneDB#

We use ov.single.run_cellphonedb_v5(...) to generate a CellPhoneDB-style communication AnnData. The downstream visualization examples below all consume this communication object directly.

cpdb_results, adata_cpdb = ov.single.run_cellphonedb_v5(
    adata,
    cpdb_file_path='./cellphonedb.zip',  # 必需参数
    celltype_key='cell_labels',
    min_cell_fraction=0.005,             # 细胞比例阈值
    min_genes=200,                       # 最小基因数
    min_cells=3,                         # 最小细胞数
    iterations=1000,                     # 置换次数
    threshold=0.1,                       # 表达阈值
    pvalue=0.05,                         # P值阈值
    threads=10,                          # 线程数
    output_dir='./cpdb_results',         # 输出目录
    cleanup_temp=True                    # 清理临时文件
)

1. Heatmap-style views with ov.pl.ccc_heatmap#

This section covers compact aggregated views, dot-matrix views, pathway-focused bubble layouts, and signaling-role summaries. It also includes a quick colormap swap so you can compare the same communication structure under a different palette.

Key constraints to keep in mind:

  • comparison_adata is only required for differential plots such as plot_type='diff_heatmap'.

  • signaling is required for pathway-specific displays such as plot_type='pathway_bubble'.

  • Aggregated heatmaps should not be filtered with interaction_use or pair_lr_use.

  • plot_type='dot' works especially well with display_by='interaction' when you want to inspect top ligand-receptor pairs.

  • Role heatmaps summarize pathway roles and are usually shown without sender_use or receiver_use filters.

2. Network views with ov.pl.ccc_network_plot#

ccc_network_plot now collects the old aggregate-circle, pathway-circle, outgoing/incoming panel, chord, ligand-receptor chord, gene chord, and differential views behind one public API. This walkthrough now also includes embedding_network, bipartite, arrow, and sigmoid examples.

Key constraints:

  • signaling is required for plot_type='pathway'.

  • pair_lr_use or interaction_use is required for plot_type='lr_chord'.

  • comparison_adata is required for plot_type='diff_network'.

  • embedding_network needs cell-type coordinates and optional background embedding points; here we reuse the original UMAP stored in adata.

  • bipartite can be focused with ligand and receptor filters.

  • arrow and sigmoid are most useful with display_by='interaction' when you want to show the sender -> ligand-receptor -> receiver flow.
