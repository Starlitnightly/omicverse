# Core Beliefs

1. The harness exists to make the agent legible and reliable, not only powerful.
2. Runtime contracts belong in code and indexed docs, not in duplicated ad hoc comments.
3. Heavy or secret-backed validation is server-only by policy.
4. Public CI may check packaging and lightweight imports, but not harness execution semantics.
5. Every meaningful agent turn should be replayable from a saved trace.
6. Framework entropy is expected; cleanup/reporting is a first-class maintenance activity.
